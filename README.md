# Лабиринт сокровищ

Учебный проект на Python — текстовая adventure-игра.
Игрок исследует лабиринт, перемещается между комнатами, собирает предметы,
решает загадки и может одержать победу, открыв сундук с сокровищами.

Проект выполнен в рамках учебного курса по Python с использованием Poetry и Git.

---

## Запуск проекта

Проект использует Poetry для управления окружением и запуском.

## Установка зависимостей:

```bash
poetry install
```

## Запуск игры

```bash
poetry run project
```

## Структура проекта
```
labyrinth_game/
├── __init__.py
├── main.py            # Точка входа и основной игровой цикл
├── constants.py       # Данные игрового мира (карта комнат)
├── player_actions.py  # Действия игрока и состояние персонажа
└── utils.py           # Вспомогательные функции (описания, загадки, победа)
```

## Архитектура и ответственность модулей

### main.py

- запуск игры
- основной игровой цикл (while not game_over)
- обработка пользовательских команд

### constants.py

- описание комнат лабиринта
- карта переходов, предметы и загадки

### player_actions.py

- перемещение игрока
- работа с предметами
- управление инвентарём

### utils.py

- вывод описаний комнат
- решение загадок
- проверка условий победы
- вспомогательные функции

## Основные механики игры

- Перемещение между комнатами (go <direction>)
- Осмотр текущей комнаты (look)
- Подбор и использование предметов (take, use)
- Просмотр инвентаря (inventory)
- Решение загадок (solve)
- Условие победы — открытие сундука с сокровищами

## Доступные команды

- go <direction> — перейти в направлении (north / south / east / west)
- look — осмотреть текущую комнату
- take <item> — поднять предмет
- use <item> — использовать предмет из инвентаря
- inventory — показать содержимое инвентаря
- solve — попытаться решить загадку или открыть сундук
- help — показать список команд
- quit — выйти из игры

## Примечание по истории коммитов

Механики этапов 2.3 (движение и предметы) и 2.4 (загадки и условие победы)
были реализованы в рамках коммита Step 2.2, так как изменения вносились
единым логическим набором без промежуточных фиксаций.
